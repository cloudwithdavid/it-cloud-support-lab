# Ticketing Lab

A Jira-based IT support lab built to demonstrate structured case handling, triage reasoning, documentation quality, and junior-level troubleshooting process.

## What This Project Demonstrates

This lab is designed to show competence in:

- support case handling
- issue triage
- structured troubleshooting
- documentation quality
- resolution vs escalation judgment
- knowledge capture through KBs and runbooks

## What This Is

This is not a Jira feature demo and not a collection of random screenshots.

It is a structured support lab where Jira is used as the system of record for working through realistic support scenarios.

## Core Skills Practiced

- turning vague user problems into structured support cases
- forming reasonable hypotheses
- choosing useful first checks
- documenting findings clearly
- resolving common issues when appropriate
- escalating with useful context when needed

## Repository Structure

- `docs/tickets/` → selected example support cases
- `docs/kb/` → knowledge base articles derived from recurring issues
- `docs/runbooks/` → operational guides and triage decision frameworks
- `docs/process/` → workflow, lifecycle, and support-process documentation
- `assets/screenshots/` → supporting visuals from the Jira lab

## Current Scope

This lab focuses on learning how to execute support work properly:

- case intake
- triage
- documentation
- troubleshooting logic
- verification
- escalation discipline

## Goal

The goal is to build and demonstrate junior-level support competence through structured case handling, not merely tool familiarity.
