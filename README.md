# IT / Cloud Support Lab

## About

This repository is a guided support-learning lab built around simulated tickets, KB articles, and runbooks. Its purpose is to practice structured support reasoning, documentation discipline, and escalation judgment across common IT and cloud support scenarios.

## What This Lab Demonstrates

- structured support case analysis and troubleshooting
- translation of vague user issues into structured cases
- repeated practice with triage, hypotheses, checks, and escalation judgment
- clear documentation of reasoning, outcomes, and lessons learned
- development of reusable support artifacts such as tickets, KBs, and runbooks

## How the Lab Is Executed

Each case is worked through in a guided loop:

1. perform first-pass reasoning on the case
2. review and tighten the reasoning through guided critique
3. question the checks, findings, resolution, and escalation logic
4. revise the ticket write-up
5. finalize the case record and capture the lesson

## Artifact Types

### Tickets

Primary artifacts used to practice issue framing, classification, hypotheses, checks, resolution vs escalation decisions, verification, and debriefing.

### KB Articles

Artifacts used to show how repeatable issue patterns can be documented for reuse.

### Runbooks

Artifacts used to show how repeated support workflows are standardized into stepwise operational guidance and escalation logic.

## Repository Structure

- `docs/process/` → workflow, lifecycle, and lab operating documents
- `docs/tickets/` → simulated support cases
- `docs/kb/` → KB articles for repeatable issue patterns
- `docs/runbooks/` → runbooks for repeatable support workflows

## Scope Boundary

This lab includes AI-supported workflow and structured reasoning. Files in this repository are intended to show support judgment, documentation structure, and troubleshooting discipline within a simulated environment rather than independent production support execution.
